## Синхронизация остатков и цен товаров с маркетплейсом Озон

Скрипт seller.py получает данные о всех товарах, загруженных в Ozon, а затем их артикулы. Скачивает остатки и цены часов с сайта поставщика Сasio. Далее происходит сопоставление остатков товаров с товарами на Ozon, определяется количество товаров в наличии. Обновленные остатки отправляются в Ozon (партиями по 100 товаров). Формируются цены на основе данных с сайта поставщика, после чего происходит обновление цен в Ozon (партиями по 900 товаров).

## Синхронизация остатков и цен товаров с Яндекс Маркетом

Скрипт market.py синхронизирует остатки и цены на товары с Яндекс Маркетом для двух типов моделей:
- FBS (Fulfillment by Seller) — модель, при которой продавец хранит товар у себя, сам его упаковывает, а маркетплейс осуществляет доставку до покупателя. 
- DBS (Delivery by Seller) — модель, где продавец самостоятельно хранит, упаковывает и доставляет товар покупателю напрямую, а маркетплейс выступает только как витрина для размещения товара. 

Сначала скрипт скачивает остатки и цены часов с сайта поставщика Сasio. Получает данные о всех FBS товарах, загруженных на Яндекс Маркет, а затем их артикулы. Далее происходит сопоставление остатков товаров с товарами на Яндекс Маркете, определяется количество товаров в наличии. Обновленные остатки отправляются в Яндекс Маркет (партиями по 2000 товаров). Формируются цены на основе данных с сайта поставщика, после чего происходит обновление цен в Яндекс Маркет (партиями по 500 товаров). То же самое повторяется для DBS товаров.
